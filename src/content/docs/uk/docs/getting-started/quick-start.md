Найшвидший спосіб спробувати OpenFishing на власному комп'ютері — це одна команда `docker run`.

## Одна команда

```bash
docker run -d -p 3000:3000 \
  -v openfishing-db:/app/data \
  -v openfishing-uploads:/app/uploads \
  ghcr.io/m1ndgames/openfishing:latest
```

Потім відкрийте **http://localhost:3000**.

Це запускає OpenFishing у режимі **єдиного орендаря, повністю відкритому** — без входу. Будь-хто, хто має доступ до
порту, має повний доступ, тож відкривайте його лише в довіреній мережі, доки не увімкнете
[автентифікацію](/uk/docs/multi-user/authentication).

## Перший вхід

- **Без `ADMIN_PASSWORD`** (як вище): застосунок відкритий — немає екрана входу, і всі
  функції доступні одразу.
- **З установленим `ADMIN_PASSWORD`**: з'являється екран входу. Ім'я користувача адміністратора завжди
  `admin`, а пароль — той, який ви встановите в `ADMIN_PASSWORD`. Обліковий запис
  повторно синхронізується зі змінної середовища під час кожного запуску.

Перегляньте [Багатокористувацький режим і автентифікація](/uk/docs/multi-user/authentication) для повної картини.

## Де зберігаються ваші дані

OpenFishing зберігає дві речі, які ви маєте зробити постійними:

| Дані | Шлях у контейнері | У наведеній вище команді |
|---|---|---|
| База даних SQLite | `/app/data` | іменований том `openfishing-db` |
| Завантажені фото | `/app/uploads` | іменований том `openfishing-uploads` |

:::caution
І база даних, і каталог завантажень **мають** бути на постійних томах. Якщо ви запускаєте
контейнер без томів, усе втрачається під час видалення контейнера.
:::

Для готового до продакшену налаштування з іменованими томами та змінними середовища у файлі використовуйте
[встановлення docker-compose](/uk/docs/getting-started/installation).
