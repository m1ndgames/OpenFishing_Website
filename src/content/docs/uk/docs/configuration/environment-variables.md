title: Змінні середовища

---

OpenFishing налаштовується повністю через змінні середовища. Кожна змінна є опційною, окрім зазначених
випадків; невстановлені змінні використовують значення за замовчуванням нижче.

## Довідкова таблиця

| Змінна | За замовчуванням | Опис |
|---|---|---|
| `DATABASE_URL` | `local.db` | Шлях до файлу бази даних SQLite. У Docker вкажіть ним на ваш том даних, напр. `/app/data/openfishing.db`. |
| `UPLOAD_PATH` | `./uploads` | Каталог для фото приманок/місць/уловів. **Змонтуйте тут том** (напр. `/app/uploads`). |
| `BASE_URL` | `http://localhost:5173` | Публічна базова URL-адреса, що використовується для побудови посилань QR-кодів і скидання пароля. Установіть її на вашу справжню HTTPS-адресу за зворотним проксі. |
| `ADMIN_PASSWORD` | _(не встановлено)_ | Установіть, щоб **увімкнути багатокористувацький вхід**. Це пароль для облікового запису `admin` (ім'я користувача завжди `admin`, повторно синхронізується під час кожного запуску). Не встановлено = повністю відкрита інстанція єдиного орендаря. |
| `AUTH_PASSWORD` | _(не встановлено)_ | **Застаріла** резервна назва для `ADMIN_PASSWORD`. Віддавайте перевагу `ADMIN_PASSWORD`. |
| `SMTP_HOST` | _(не встановлено)_ | Ім'я хоста SMTP-сервера для листів «забули пароль». |
| `SMTP_PORT` | _(не встановлено)_ | Порт SMTP-сервера. |
| `SMTP_SECURE` | _(не встановлено)_ | Використовувати TLS для SMTP-з'єднання. |
| `SMTP_USER` | _(не встановлено)_ | Ім'я користувача SMTP. |
| `SMTP_PASS` | _(не встановлено)_ | Пароль SMTP. |
| `SMTP_FROM` | _(не встановлено)_ | Адреса `From:` для вихідної пошти. |
| `DEMO_MODE` | _(не встановлено)_ | Будь-яке значення = **демо-режим лише для читання** (усі записи заблоковані). |
| `CHATBOT` | _(не встановлено)_ | Будь-яке істинне значення вмикає **ШІ-чат-бот** (потребує змінних LiteLLM). |
| `LITELLM_URL` | _(не встановлено)_ | Базова URL-адреса вашого проксі LiteLLM, напр. `http://litellm:4000`. |
| `LITELLM_MODEL` | _(не встановлено)_ | Назва моделі, що відповідає запису в `litellm.config.yaml`. |
| `LITELLM_VISION_MODEL` | _(не встановлено)_ | Опційна модель із підтримкою зору для розпізнавання риби/приманок. Повертається до `LITELLM_MODEL`. |
| `BODY_SIZE_LIMIT` | `104857600` | Максимальний розмір завантаження в байтах (за замовчуванням ≈ 100 МБ). |

:::note
Функція SMTP «забули пароль» з'являється лише тоді, коли встановлено принаймні `SMTP_HOST` **і** `SMTP_FROM`.
Без них опція скидання прихована.
:::

## Примітки щодо значень за замовчуванням

Наведені вище значення за замовчуванням — це вбудовані в код типові значення застосунку. У контейнері ви
зазвичай перевизначаєте шляхи, щоб вони вказували на змонтовані томи:

```bash
DATABASE_URL=/app/data/openfishing.db
UPLOAD_PATH=/app/uploads
```

## Згруповано за призначенням

- **Сховище:** `DATABASE_URL`, `UPLOAD_PATH`, `BODY_SIZE_LIMIT`
- **Мережа / посилання:** `BASE_URL`
- **Автентифікація:** `ADMIN_PASSWORD` (`AUTH_PASSWORD` застаріле)
- **Пошта / скидання пароля:** `SMTP_HOST`, `SMTP_PORT`, `SMTP_SECURE`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM`
- **Функції ШІ:** `CHATBOT`, `LITELLM_URL`, `LITELLM_MODEL`, `LITELLM_VISION_MODEL`
- **Демо:** `DEMO_MODE`

Див. [Багатокористувацький режим і автентифікація](/uk/docs/multi-user/authentication),
[Функції ШІ](/uk/docs/ai/ai-features) та [Демо-режим](/uk/docs/demo/demo-mode), щоб зрозуміти, як ці групи
поєднуються.
